Wstęp

Program Split służy do dzielenia tekstu na zdania. Można to robić przy pomocy prostych wbudowanych reguł jak i za pomocą zewnętrznego pliku SRX zawierającego reguły podziału.

Wymagania

Do uruchomienia programu potrzebna jest Java Runtime Environment w wersji 1.5. Do zbudowania potrzeba Java Software Development Kit w wersji 1.5 oraz narzędzia Ant. Do testowania konieczna jest biblioteka JUnit 3.8 w ścieżce CLASSPATH. System operacyjny w zasadzie dowolny, testowano w systemach Linux i Windows.

Instalacja

Programu nie trzeba instalować, wystarczy go rozpakować.

Uruchamianie

Do uruchomienia programu służy skrypt split w katalogu bin. Np. będąc w głównym katalogu projektu w systemie Linux uruchamia się go następująco:
bin/split
w systemie Windows natomiast w wierszu poleceń wydaj polecenie:
bin\split
Program można też uruchomić bezpośrednio za pomocą Javy. W systemie typu Unix wygląda to tak:
java -cp "$CLASSPATH:$<katalog-projektu>/lib/split.jar" Split
a w Windows tak:
java -cp "%CLASSPATH%;$<katalog-projektu>/lib/split.jar" Split
Program przyjmuje kilka argumentów o których informacje można uzyskać uruchamiając go z opcją -h:
split -h
Bez argumentów program czyta tekst ze standardowego wyjścia i dzieli go na segmenty za pomocą wbudowanych reguł, a wynik wypisuje na standardowe wyjście, po 1 segmencie na linię. Najczęściej używanym wywołaniem jest zapewne:
split -s plik-srx < plik-wejsciowy > plik-wyjściowy
które dzieli zadany plik wejściowy za pomocą domyślnej reguły z pliku SRX i wynik zapisuje w danym pliku wyjściowym. Do ustalenia języka tekstu (i przez to zastosowanie odpowiedniej reguły SRX) służy opcja -l. Żeby wybrać inną niż domyślna regułę mapującą język na regułę podziału trzeba użyć opcji -m. Można też bezpośrednio wymusić stosowanie konkretnej reguły podziału opcją -r. Do kontrolowania wyjścia służą opcje -b i -e ustawiające odpowiednio napis wypisywany przed segmentem i po segmencie.

Badanie wydajności

Do badania wydajności służy skrypt performance w katalogu bin. Aby uzyskać pomoc trzeba użyć opcji -h. Działanie tego skryptu polega na wygenerowaniu losowego tekstu o zadanej (za pomocą opcji -t) długości w znakach i podzieleniu go za pomocą wygenerowanych reguł SRX (ilość reguł ustawia się opcją -r, a ilość wyjątków w pojedynczej regule - jej długość - opcją -l). Jako wynik wyświetlany jest czas działania programu.

Testowanie

Żeby uruchomić testy modułowe zawarte w kodzie uruchom skrypt test z katalogu bin (w podobny sposób jak skrypt split). Do jego uruchomienia potrzeba biblioteki JUnit w zmiennej CLASSPATH.

Budowanie

Jeśli posiadasz wersje binarną tego programu to jest on już skompilowany. W przeciwnym wypadku możesz użyć narzędzia Ant. Aby skompilować projekt, w głównym katalogu projektu wydaj polecenie:
ant
Istnieje więcej celów służących do generowania dystrybucji binarnej (bindist), źródłowej (srcdist), dokumentacji (javadoc). Aby uzyskać pełną listę celów Ant-a w głównym katalogu projektu wydaj polecenie:
ant -p

Formaty danych

Wejście

Tekst zakodowany w kodowaniu systemowym.

Wyjście

Każdy segment rozpoczyna się napisem ustalonym opcją -b (domyślnie <s>), a kończy napisem ustalonym opcją -e (domyślnie </s>) i znakiem nowego wiersza. Tekst będzie zakodowany kodowaniem systemowym. Uwaga w Windows-ach kodowanie w konsoli jest różne od systemowego, dlaetego w konsoli wyskakują 'krzaczki', ale w innych programach znaki powinny być poprawne.

Pliki SRX

Dokument XML. posiada nagłówek <header> (na razie ignorowany) i ciało <body>. W ciele znajdują się reguły języka i reguły mapowania. Reguły języka zawierają reguły podziału (break="yes") i wyjątki (break="no") składające się z dwóch wyrażeń regularnych - przed i po znaku podziału. Reguły mapujące określają jak kody języka (zapisane jako wyrażenie regularne) przekładają się na reguły języka. Pełna specyfikacja znajduje się na stronie www.lisa.org. Przykładowe pliki znajdują się w katalogu examples.
